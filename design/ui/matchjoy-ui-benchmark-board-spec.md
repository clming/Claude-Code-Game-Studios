# MatchJoy UI Benchmark Board Spec

> Status: Draft
> Last Updated: 2026-03-30
> Purpose: Define how to build the first benchmark board for MatchJoy UI art design.

## Goal

The benchmark board is not a scrapbook.

It exists to answer:

1. How should the board be framed as the visual center?
2. How should HUD information be layered without clutter?
3. How should result panels feel branded instead of generic?
4. How should urgency, reward, and polish be expressed?

## Tool

Primary tool:

- PureRef for the raw benchmark board

Secondary tool:

- FigJam or Figma for annotated presentation boards

## Board Structure

Create one board with six sections.

### 1. Board Framing

Collect examples of:

- board containers
- outer frames
- top labels
- side status chips
- board-to-background separation

Questions:

- What makes the board feel premium?
- What keeps the board readable while still decorative?

### 2. HUD Layering

Collect examples of:

- move counters
- goal cards
- top bars
- order cards
- chapter / level tags

Questions:

- What information is large?
- What information is compact?
- How do they separate metadata from gameplay-critical data?

### 3. Results / Reward Panels

Collect examples of:

- win panels
- fail panels
- star result layouts
- completion cards
- reward copy hierarchy

Questions:

- What feels like a celebration?
- What feels like a branded failure state rather than a system alert?

### 4. Art Language

Collect examples of:

- panel materials
- icon style
- badge style
- borders
- highlights
- glows

Questions:

- Is the style flat, glossy, toy-like, paper-like, or candy-coated?
- Which treatment best fits "Candy Workshop Counter"?

### 5. Motion / FX References

Collect examples of:

- panel entrance motion
- star reveal motion
- badge motion
- reward burst motion
- lightweight sparkles / glow accents

Questions:

- What motion feels satisfying but not noisy?
- What level of FX polish is appropriate for this prototype?

### 6. Anti-Reference

Collect a small set of examples to avoid.

Examples:

- overly generic mobile puzzle UI
- purple-heavy default gradients
- dashboard-like corporate cards
- over-busy casino FX

Questions:

- What would make MatchJoy lose its own identity?

## Annotation Rules

Every benchmark image should have at least one note.

Allowed note categories:

- `Keep`
- `Adapt`
- `Avoid`
- `Motion`
- `Material`
- `Hierarchy`

Example note:

- `Adapt: strong board frame, but reduce metallic contrast`
- `Keep: readable move count hierarchy`
- `Avoid: reward panel is too close to casino UI`

## Deliverables

The first benchmark pass should produce:

1. one PureRef board,
2. one annotated export,
3. one short written summary,
4. one shortlist of 2-3 likely visual routes.

## Exit Criteria

Do not move into high-fidelity UI design until the board can clearly answer:

1. which benchmark influences board framing,
2. which benchmark influences HUD hierarchy,
3. which benchmark influences result polish,
4. which visual patterns are explicitly rejected.
