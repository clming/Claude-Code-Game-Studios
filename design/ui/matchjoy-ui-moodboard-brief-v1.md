# MatchJoy UI Moodboard Brief v1

> Status: Draft
> Last Updated: 2026-03-30
> Purpose: Define the first-round mood board and visual exploration task for MatchJoy UI art.

## Objective

Create a mood board that explores 3 candidate visual routes for the playable single-level slice.

The mood board should help the team choose:

- color system,
- panel material direction,
- typography tone,
- ornament language,
- UI motion tone,
- overall polish target.

## Core Product Read

The game should read as:

- bright,
- crafted,
- premium,
- candy-forward,
- workshop-themed,
- readable on mobile.

It should not read as:

- debug UI,
- generic puzzle clone UI,
- casino overload,
- corporate dashboard.

## Candidate Routes

The first mood board should explore these 3 routes.

### Route A. Candy Workshop Counter

Keywords:

- warm
- handcrafted
- frosted packaging
- bakery ticket
- syrup accents

Best for:

- current product framing
- readable premium casual tone

Look for:

- laminated paper textures
- rounded cards
- caramel and sugar tones
- premium confection counters

### Route B. Playful Candy Cabinet

Keywords:

- toy-like
- chunky
- bright
- collectible
- cheerful

Best for:

- stronger broad-audience friendliness
- more obvious casual-game readability

Look for:

- large soft buttons
- toy packaging
- glossy rounded iconography
- bold color blocking

### Route C. Boutique Dessert Lab

Keywords:

- elegant
- modern
- upscale
- plated sweets
- restrained shine

Best for:

- more premium market positioning
- cleaner hero-board presentation

Look for:

- cream marble tones
- menu-card hierarchy
- refined gold accents
- controlled glow and polish

## Required Moodboard Sections

Each route should include:

1. color swatches
2. type direction
3. panel / material samples
4. icon and badge samples
5. motion / FX keywords
6. board-frame inspiration
7. HUD-card inspiration
8. result-card inspiration

## Evaluation Questions

For each route, answer:

1. Does the board remain the visual hero?
2. Does the HUD stay readable on a phone-sized frame?
3. Does the result panel feel branded?
4. Can this route support pressure states without becoming ugly?
5. Can this route support stars, goals, badges, and candy icons coherently?

## Expected Output

The first-pass mood board review should end with:

1. one recommended route,
2. one backup route,
3. one rejected route,
4. one list of "must keep" traits for the chosen direction.

## Follow-up Deliverables

Once the route is chosen, the next design tasks are:

1. HUD high-fidelity mockup
2. Board shell high-fidelity mockup
3. Results panel high-fidelity mockup
4. component inventory
5. asset production list

## Notes For Engineering

Current Unity-side UI scripts are scaffolding only.

They already give us:

- screen regions,
- information buckets,
- runtime states,
- some state-dependent labels.

They do not define final art direction.

That direction must now come from the mood board and design files, then be integrated back into Unity.
